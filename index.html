Error Debugger — what each error means
❌ IndentationError
IndentationError: expected an indented block / unexpected indent
Cause: Wrong or inconsistent spaces/tabs for indentation.
Fix: Use 4 spaces consistently. Never mix tabs and spaces.
# WRONG
def my_func():
print('hi')   -- ❌ not indented
# CORRECT
def my_func():
    print('hi')   -- ✅ 4 spaces
❌ NameError
NameError: name 'x' is not defined
Cause: Using a variable before assigning it, or typo in variable name.
Fix: Define the variable before using it. Check spelling.
# WRONG — using before defining
print(result)   -- ❌ result not defined yet
result = 10
# CORRECT
result = 10
print(result)   -- ✅
❌ TypeError
TypeError: can only concatenate str (not "int") to str
Cause: Mixing incompatible types — like adding string and integer.
Fix: Convert types explicitly using str(), int(), float().
# WRONG
print("Score: " + 90)     -- ❌ str + int
# CORRECT
print("Score: " + str(90))  -- ✅ convert to str
print(f"Score: {90}")       -- ✅ f-string (better)
❌ IndexError
IndexError: list index out of range
Cause: Accessing index that doesn't exist. List of 3 items has indices 0,1,2 only.
Fix: Check len() before accessing. Use -1 for last element.
# WRONG
arr = [1, 2, 3]
arr[3]   -- ❌ index 3 doesn't exist (max is 2)
# CORRECT
arr[2]    -- ✅ last element
arr[-1]   -- ✅ last element shortcut
❌ KeyError
KeyError: 'name'
Cause: Accessing dict key that doesn't exist.
Fix: Use .get() which returns None instead of error, or check key exists first.
# WRONG
d = {'age': 25}
d['name']         -- ❌ 'name' key doesn't exist
# CORRECT
d.get('name')      -- ✅ returns None if missing
d.get('name', 'N/A') -- ✅ returns 'N/A' if missing
❌ AttributeError
AttributeError: 'list' object has no attribute 'upper'
Cause: Calling a method that doesn't belong to that data type.
Fix: Check data type first. .upper() is for strings, not lists.
# WRONG
my_list = ['a', 'b']
my_list.upper()   -- ❌ lists don't have .upper()
# CORRECT
my_str = 'hello'
my_str.upper()    -- ✅ strings have .upper()
❌ SyntaxError — = vs ==
SyntaxError: invalid syntax
Cause: Using single = (assignment) instead of == (comparison) in if statement.
Fix: Always use == for comparison.
# WRONG
if x = 5:    -- ❌ = is assignment, not comparison
# CORRECT
if x == 5:   -- ✅ == is comparison
❌ Return inside loop
No error — but function returns too early (only processes first element)
Cause: return inside for loop exits function on first iteration.
Fix: Update a variable inside the loop. return AFTER the loop finishes.
# WRONG — returns after first vowel only
for i in s:
    if i in vowels:
        return count + 1   -- ❌ exits on first vowel
# CORRECT — count all, return after loop
for i in s:
    if i in vowels:
        count += 1   -- ✅ accumulate
return count          -- ✅ return after loop
❌ Mutable default argument
No error — but function behaves strangely across calls
Cause: Using a mutable object like [] as default parameter. It's shared across all calls.
Fix: Use None as default, create the object inside the function.
# WRONG — list is shared across all calls!
def add_item(item, lst=[]):
    lst.append(item)
    return lst
# CORRECT
def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

Maximum in array — without max()
def find_max(arr):
    result = arr[0]   # start with first element
    for i in arr:
        if i > result:
            result = i
    return result
Second largest — without sorted()
def second_largest(arr):
    first = second = float('-inf')
    for i in arr:
        if i > first:
            second = first   # old first drops down
            first = i
        elif first > i > second:
            second = i
    return second
# float('-inf') = negative infinity, any number beats it
# handles duplicates: [5,5,3] returns 3 not 5

Count vowels
def count_vowels(s):
    vowels = 'aeiou'
    count = 0
    for char in s.lower():
        if char in vowels:
            count += 1
    return count   # return AFTER loop, never inside

# One-liner version
def count_vowels(s):
    return sum(1 for c in s.lower() if c in 'aeiou')
Factorial
def factorial(n):
    result = 1
    for i in range(1, n+1):   # n+1 to include n itself!
        result *= i
    return result
# factorial(5) = 1×2×3×4×5 = 120
Fibonacci sequence
def fibonacci(n):
    a, b = 0, 1
    result = []
    for _ in range(n):
        result.append(a)
        a, b = b, a + b   # swap in one line
    return result
# fibonacci(6) = [0,1,1,2,3,5]

Remove duplicates from list — preserve order
def remove_duplicates(lst):
    seen = set()
    result = []
    for item in lst:
        if item not in seen:
            result.append(item)
            seen.add(item)
    return result
# [1,2,2,3,1] → [1,2,3]

Check prime number
def is_prime(n):
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):  # only check up to √n
        if n % i == 0:
            return False
    return True

filter — keep only matching elements
numbers = [1, 2, 3, 4, 5, 6]

list(filter(lambda x: x % 2 != 0, numbers))  # [1,3,5] odds

sorted — sort by custom condition

How Python behaves — tricky things
Mutable vs immutable
# Immutable: int, str, tuple — cannot change in-place
x = 'hello'
x.upper()   # doesn't change x!
x = x.upper()  # must reassign

# Mutable: list, dict, set — change in-place
lst = [1, 2, 3]
lst.append(4)  # modifies lst directly — no reassign needed
Copy vs reference
# Lists are reference types — assignment shares the same list
a = [1, 2, 3]
b = a          # b points to SAME list
b.append(4)   # a is now [1,2,3,4] too!

# To copy
b = a.copy()   # shallow copy
b = a[:]       # also shallow copy
Integer division behavior
10 / 3     # 3.333... — float division
10 // 3    # 3 — floor division (truncates)
10 % 3     # 1 — remainder
-7 // 2    # -4 — floors toward negative infinity!
-7 % 2     # 1 — not -1 (Python modulus always positive)

'' (empty string)Any non-empty string
[] (empty list)Any non-empty list
{} (empty dict)Any non-empty dict
